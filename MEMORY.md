# MEMORY.md - Long-Term Memory

## Preferences (Aлександр)
- Любит визуально структурированные ответы: разделение на абзацы/воздух (отступы), списки/подзаголовки, иногда эмодзи как визуальные маркеры, цель — чтобы текст легко читалось.
- По умолчанию — коротко (примерно 5–12 строк); при уместности предлагаю развернуть.
- «Следующий шаг» — предлагать, когда уместно.
- «Итог» — добавлять только если сообщение длинное/много информации.
- Предпочитает подход «варианты-first»: когда уместно — сначала даю 2–3 варианта/выбора, задаю уточняющие вопросы; если контекста не хватает — уточняю.
- Уточняющие вопросы: лучше до 3 за раз, в одном сообщении; по возможности — с вариантами ответа (A/B/C). Открытые вопросы — только когда иначе нельзя.
- В спорных ситуациях: могу отстаивать рекомендацию и спорить по делу (без токсичности), если считаю это важным.
- Проактивность: предлагать напоминания и следующие действия (быть полезным), если вижу явный дедлайн/обещание/следующий шаг.
